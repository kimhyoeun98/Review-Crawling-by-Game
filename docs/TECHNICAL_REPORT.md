# 테크니컬 리포트 — Steam 리뷰 감성 분석 서비스

> KcELECTRA 딥러닝 기반 게임 리뷰 본문 감성 분석 및 부정 리뷰 필터링 서비스

---

## 1. 프로젝트 개요

### 1.1 배경 및 목표
Steam 게임 리뷰는 `추천 / 비추천`이라는 **이진 평점**을 제공하지만, 이 평점만으로는 다음을 알기 어렵다.

- 리뷰 **본문**이 실제로 긍정적인지 부정적인지 (평점과 본문이 다를 수 있음)
- 장르(카테고리)별 감성 경향의 차이
- 작성 중인 리뷰가 지나치게 부정적인지 (실시간 판별)

본 프로젝트는 Steam 리뷰를 대량 수집하여 한국어 감성 분석 모델(KcELECTRA)을 학습하고, 이를 통해 위 문제들을 해결하는 **웹 서비스**를 구축하는 것을 목표로 한다.

### 1.2 핵심 결과
- 560,520건의 한국어 리뷰 수집 (7개 액션 장르)
- KcELECTRA 파인튜닝 → **검증 정확도 89.3%** (TF-IDF 87.2%, LSTM 85.7% 대비 우수)
- 전체 리뷰의 **13.9%가 평점과 본문 감성이 불일치** → 본문 분석의 필요성 입증
- 통계 · 실시간 · 감성검사 3개 기능의 Streamlit 서비스 구현

## 2. 시스템 아키텍처

```
[ 데이터 수집 ]          [ 모델 / 분석 ]              [ 서비스 ]
 lib/                     train_kcelectra.py           service/
  ├ SteamAppIDExtractor    → model/kcelectra/           ├ SentimentAnalyzer (추론·차단)
  ├ SteamReviewCrawler    predict_all_sentiments.py     ├ analytics  (본문 감성 통계)
  └ ReviewPreprocessor     → *_scored.jsonl             ├ live_stats (Steam 실시간 API)
        │                  build_category_map.py        └ app.py     (Streamlit UI)
        ▼                   → game_category_map.json            │
   data/steam_raw_reviews.jsonl ───────────────────────────────┘
```

설계 원칙:
- **단일 진입점 + 클래스 모듈**: `main.py`가 `lib/`의 클래스들을 호출하는 파이프라인 구조
- **수집(lib) / 서비스(service) 분리**: 역할별 패키지로 응집도 확보
- **경로 일원화**: 모든 데이터·모델 경로를 루트 `data/`, `model/` 기준으로 통일

## 3. 데이터 수집 (크롤링)

### 3.1 수집 파이프라인
1. **`SteamAppIDExtractor`** (Selenium): 장르 카테고리 페이지에서 '최고 인기 제품' 기준으로 게임 appid 목록을 무한 스크롤로 추출 → `data/steam_appid/target_appids_<장르>.jsonl`
2. **`SteamMassiveReviewCrawler`** (Steam API): 각 appid의 한국어 리뷰를 `cursor` 페이지네이션으로 누락 없이 수집
3. **체크포인트**(`completed_appids.txt`)로 중복 방지 및 이어받기 지원

### 3.2 수집 규모
- 총 **560,520건**, 고유 게임 533개, **7개 장르**
  (FPS / TPS / 핵앤슬래시 / 리듬·아케이드 / 런앤점프 / 슈팅(shmup) / 격투)

### 3.3 레코드 스키마
```json
{"appid": "578080", "게임 이름": "PUBG: BATTLEGROUNDS",
 "카테고리": "action_fps", "작성자": "7656...", "리뷰 내용": "...", "평점": "추천"}
```

## 4. 데이터 전처리

`ReviewPreprocessor` / 학습 스크립트에서 수행:
- **정제**: 제어문자·연속 공백 정규화 (`\r\n\xa0` → 공백)
- **이진 라벨링**: 추천=1, 비추천=0
- **중복 제거**: (게임, 작성자, 본문) 기준
- **클래스 균형 샘플링**: 학습 시 추천/비추천 각 30,000건씩 균등 추출 (불균형 방지)

> 비속어·신조어는 **별도 제거하지 않는다.** KcELECTRA가 비속어 포함 한국어 댓글로 사전학습되어 있어, 비속어를 문맥 속에서 그대로 해석하기 때문이다.

## 5. 모델 학습 및 비교 실험

감성 분류 모델은 단순한 방식부터 단계적으로 시도하며 성능을 비교했다. 모든 실험은 **동일한 60,000건 데이터(추천/비추천 각 30,000 균형)와 동일한 검증 분할**(학습 90% / 검증 10%, stratified, `random_state=42`)로 진행해 공정하게 비교했다.

### 5.1 1차 시도 — TF-IDF + LogisticRegression
가장 가볍고 빠른 전통적 머신러닝 기법으로 출발했다.
- **구성**: char_wb n-gram(2~4) TF-IDF 벡터화 + LogisticRegression
- **장점**: 형태소 분석 없이도 빠르게 학습, 한국어에 비교적 강건
- **결과**: 검증 정확도 **87.2%** (Macro F1 0.872)
- **한계**: 단어 표면형의 빈도 기반이라 어순·문맥·미묘한 감성(반어, 역접)을 포착하지 못함

### 5.2 2차 시도 — LSTM (Okt + Embedding + LSTM)
순서(문맥)를 반영하기 위해 딥러닝 순환신경망을 도입했다.
- **구성**: Okt 형태소 토큰화 → Embedding(100차원) → LSTM(64) → Dense(sigmoid)
- **결과**: 검증 정확도 **85.7%** (Macro F1 0.857)
- **한계**: 학습 정확도는 약 90%까지 올랐지만 검증은 ~85.7%에서 정체 → **과적합**. 6만 건 규모의 데이터로는 임베딩과 LSTM 가중치를 처음부터 학습하기에 부족했다.

### 5.3 3차 시도 — KcELECTRA (최종 채택)
대규모 사전학습 트랜스포머를 파인튜닝했다.
- **모델**: `beomi/kcelectra-base-v2022` — 네이버 뉴스 댓글 등 **구어체·비속어가 풍부한 한국어 데이터로 사전학습**된 ELECTRA. 게임 리뷰의 비속어·신조어를 사전이 아닌 **문맥**으로 이해.
- **학습 설정**: 최대 길이 128 / 배치 64 / 2 epoch / LR 2e-5 / FP16 혼합정밀 + 동적 패딩 (RTX A4000)
- **결과**: 검증 정확도 **89.3%** (Macro F1 0.893)

### 5.4 비교 결과 (동일 검증셋 6,000건)
| 시도 | 방식 | 검증 정확도 | Macro F1 |
|:---:|------|:----------:|:--------:|
| 1차 | TF-IDF(char 2~4) + LogisticRegression | 87.2% | 0.872 |
| 2차 | LSTM (Okt + Embedding + LSTM) | 85.7% | 0.857 |
| **3차** | **KcELECTRA (채택)** | **89.3%** | **0.893** |

> 흥미롭게도 LSTM이 가장 낮았다. 데이터 규모가 제한적일 때는 처음부터 학습하는 LSTM보다, 방대한 한국어 텍스트로 **사전학습된** KcELECTRA를 파인튜닝하는 편이 일반화 성능에서 크게 유리했다.

### 5.4 KcELECTRA 상세 검증 지표 (검증 6,000건)
| 클래스 | 정밀도 | 재현율 | F1 |
|--------|:-----:|:-----:|:--:|
| 긍정(추천) | 0.912 | 0.870 | 0.890 |
| 부정(비추천) | 0.876 | 0.916 | 0.895 |

**혼동행렬** (행=실제, 열=예측)
| | 예측 부정 | 예측 긍정 |
|---|:---:|:---:|
| 실제 부정 | 2749 | 251 |
| 실제 긍정 | 391 | 2609 |

> 부정 재현율이 0.916으로 높아, **부정 리뷰 차단** 기능에 적합하다.

## 6. 전체 리뷰 분석 및 핵심 인사이트

학습된 모델로 전체 560,520건을 일괄 추론(`predict_all_sentiments.py`)하여 본문 감성을 캐시(`*_scored.jsonl`)했다.

### 핵심 발견 — 평점 ≠ 본문 감성
- 전체의 **13.9% (77,795건)** 가 평점과 모델의 본문 감성 판정이 **불일치**
- 예: *"진짜 어렵고 짜증나지만 그래도 추천"* (평점 추천 / 본문 부정),
      *"재밌긴 한데 과금 때문에 비추천"* (평점 비추천 / 본문 긍정)

→ 따라서 통계의 긍정/부정을 **평점이 아닌 모델의 본문 감성 기준**으로 산출하도록 설계했다.

## 8. 어려웠던 점 및 해결 과정

### 8.1 카테고리 '기타'가 73.8% — 이름 매칭 실패
- **문제**: 리뷰 레코드에 `appid`가 없어 게임명 문자열로만 카테고리를 매칭했는데, 매핑이 불완전(201개)해 리뷰의 73.8%(PUBG 20만건 포함)가 '기타'로 분류됨.
- **원인**: 리뷰 게임명 ↔ 카테고리 맵 게임명을 잇는 공통 키 부재 (`steam_appid`는 appid만, 리뷰는 이름만 보유).
- **해결**:
  1. 매핑 빌드 스크립트에 재시도 로직 추가 후 재실행 → 매핑 201 → 352개, **기타 73.8% → 10.8%**
  2. 크롤러를 수정해 리뷰 레코드에 `appid`·`카테고리`를 직접 저장 → 향후 이름 매칭 불필요

### 8.2 모델 정확도 한계
- **문제**: TF-IDF·LSTM으로는 한국어 리뷰의 미묘한 감성·비속어 맥락 포착이 약함.
- **해결**: 한국어 댓글로 사전학습된 KcELECTRA 파인튜닝 → 89.3% (비교 실험으로 우위 입증).

### 8.3 평점과 본문 감성의 불일치
- **문제**: 평점 집계만으로는 본문이 실제로 긍정인지 알 수 없음.
- **해결**: 전체 리뷰를 모델로 전수 분석하여 통계를 본문 감성 기준으로 전환, 불일치율 13.9% 정량화.

### 8.4 학습 속도 / 환경
- **문제**: 초기 설정으로 학습에 수 시간 예상. torch CPU 버전 설치로 GPU 미사용.
- **해결**: CUDA torch 재설치 + **FP16 혼합정밀 + 동적 패딩 + 배치 확대**로 수 분으로 단축 (GPU 사용률 98%).

### 8.5 한글 출력 인코딩 크래시
- **문제**: Windows 콘솔(cp949)이 유니코드(✓ 등) 출력 중 비정상 종료.
- **해결**: `sys.stdout.reconfigure(encoding='utf-8')`로 학습 완주.

### 8.6 Okt(KoNLPy) Java 힙 부족
- **문제**: LSTM 비교 실험 중 5.4만 건 형태소 분석에서 JVM `OutOfMemoryError`.
- **해결**: `konlpy.jvm.init_jvm(max_heap_size=4096)`로 힙 상향.

### 8.7 프로젝트 구조 정리
- **문제**: 중첩 git 저장소와 루트 중복본이 공존하여 코드/데이터가 산재.
- **해결**: 단일 저장소로 통합, `lib`(수집)/`service`(서비스) 모듈 구조 확립, 모든 경로 일원화.

## 9. 결론 및 향후 계획

### 9.1 결론
- Steam 리뷰 크롤링부터 딥러닝 감성 분석, 웹 서비스까지 **엔드투엔드 파이프라인**을 구축했다.
- KcELECTRA가 전통적 ML/DL 대비 정확도와 일반화에서 우위를 보였고, 특히 **평점과 본문 감성의 13.9% 불일치**를 통해 본문 기반 감성 분석의 필요성을 데이터로 입증했다.

### 9.2 향후 계획
- **중립 클래스** 추가로 '보통' 리뷰 구분 (현재 이진 분류)
- `appid` 기반 전체 재크롤링으로 데이터 확대 및 카테고리 정확도 향상
- 카테고리 매핑 보강 및 차단 임계값 사용자 튜닝
